# Releasing GoldenEye-Recomp

The playable artifacts (`ge`, the APK) embed PPC code generated by `rexglue codegen`
from your own GoldenEye 007 XEX, so they **cannot** be built in public CI — the GitHub
Actions workflow is compile-verify only. Releases are built locally and uploaded with
`gh`. `scripts/cut-release.sh` automates that.

## One-time setup: release keystore

The Android release APK is signed with a local keystore that is **never committed**
(`*.jks` and `android/keystore.properties` are gitignored). Back these up somewhere
safe — losing the key blocks future signed updates to the same app identity.

```sh
cd android
PW="<a-strong-password>"
keytool -genkeypair -v -keystore release.jks -alias goldeneye \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -storepass "$PW" -keypass "$PW" \
  -dname "CN=GoldenEye Recomp, O=jeffory, C=US"
cat > keystore.properties <<EOF
storeFile=release.jks
storePassword=$PW
keyAlias=goldeneye
keyPassword=$PW
EOF
chmod 600 release.jks keystore.properties
```

`android/app/build.gradle` picks this up automatically; when the props file is absent
(e.g. CI) signing is skipped so configuration still succeeds.

## JDK note (important)

The Gradle 8.9 wrapper cannot compile build scripts under Java 23+. If your system
default `java` is newer (e.g. Java 25), builds only succeed while the compiled build
script is cached — the first `build.gradle` change (which a release performs) breaks with
`Unsupported class file major version`. `scripts/cut-release.sh` therefore auto-pins
`JAVA_HOME` to a compatible JDK (8–22), preferring Gradle's auto-provisioned Adoptium
toolchain under `~/.gradle/jdks/` (e.g. `eclipse_adoptium-17-amd64-linux.*`), then
`/usr/lib/jvm/java-{17,21}-openjdk`. If none exists, install one
(`sudo dnf install java-21-openjdk-devel`) or open the project once in Android Studio to
let it provision a JBR/toolchain.

## Cut a release

Prerequisites: `gh auth login` done; `generated/` present (codegen run); both repos at the
commits you want to ship.

```sh
# prerelease (default)
scripts/cut-release.sh v1.3.0-android.1

# full release
scripts/cut-release.sh v1.3.0 --stable
```

The script bumps `versionName`/`versionCode`, commits, pushes both repos + an annotated
tag, builds the **signed APK** and the **Linux amd64 bundle**, generates notes (changelog +
pinned game/SDK SHAs), and publishes the GitHub release with both assets attached.

Flags: `--stable` (full release), `--allow-dirty`, `--sdk DIR` (SDK checkout path).

## Verify

```sh
apksigner verify --print-certs dist/GoldenEye-Recomp-v1.3.0-android.1-android-arm64.apk
gh release view v1.3.0-android.1
```
